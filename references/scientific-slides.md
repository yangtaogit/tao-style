# Scientific Slides

Status: first active Tao Style module for scientific reports and slide decks.

## Scope

Use this reference when Tao asks for a scientific slide report, research presentation, seminar slides, group meeting slides, defense-style slides, or a Beamer report.

## Output Format Protocol

- When Tao asks to generate a slides report or scientific presentation and has not specified a format, ask whether to use Beamer.
- If Tao says yes or explicitly asks for Beamer, use the Beamer workflow below.
- If Tao says no, use the requested format, such as PowerPoint, Markdown slides, or another tool, while keeping confirmed Tao Style visual preferences where practical.
- If the user already gives a template, venue requirement, or target format, follow that instruction directly.

## Beamer Template

- Use `yangtaogit/tao-slides` as Tao's preferred Beamer template source: `https://github.com/yangtaogit/tao-slides`.
- Before generating a Beamer report, retrieve or locate the template, inspect its README, source files, examples, theme files, and build commands, then follow the actual template structure.
- Do not invent template file names, class names, theme names, Makefile targets, or package options when the template is unavailable.
- If network access is unavailable and no local copy of the template exists, ask Tao for the template files or permission to proceed with a minimal temporary Beamer file.
- Work in a copy or a new project directory rather than editing the template source in place, unless Tao explicitly asks to modify the template.
- Preserve the template's macros, theme settings, fonts, colors, logos, and layout unless Tao asks to change them.
- Compile with the template's documented command when possible. If no command is documented, inspect the project for `Makefile`, `latexmkrc`, build scripts, or main `.tex` files before choosing a compiler.

## Slide Content Style

- Prefer a clear research narrative: motivation, problem, method, key results, implications, and next steps.
- Keep each slide focused on one main message.
- Use concise slide text; move derivations, implementation details, or long explanations to backup slides when needed.
- Use section or transition slides only when they help the audience follow the report structure.
- Keep equations readable and avoid dense multi-line derivations on main-result slides.
- Keep code snippets short and include them only when implementation details matter.

## Figures and Tables

- Use Tao Style scientific plotting rules for newly generated plots unless Tao asks otherwise.
- Prefer vector figures for plots and diagrams in Beamer.
- Keep figure labels, units, legends, and annotations readable at slide size.
- Use `Quantity [Unit]` axis label format for scientific plots in slides.
- Avoid overcrowding slides with many panels; split across slides or use a staged explanation when comparison would become hard to read.
- For tables, prefer compact tables that support the slide's main message; move full tables to backup slides.

## Beamer QA Checklist

- The deck compiles without LaTeX errors.
- Slides do not have overfull text boxes, clipped figures, or unreadable labels.
- The title, author, affiliation, date, and section structure match Tao's request.
- All figures are included from stable relative paths.
- Bibliography and citations compile if references are used.
- The final output includes the source `.tex` project and a compiled PDF when the environment supports compilation.
